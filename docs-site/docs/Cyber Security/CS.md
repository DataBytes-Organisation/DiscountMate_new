# DiscountMate Cyber Security Handover

## Section Overview

This section provides a handover summary for the cyber security work completed on the DiscountMate backend during this study period.

The purpose of this documentation is to help the next developer or team understand what security work has already been completed, where the main files are located, and what security items still need to be reviewed in the next study period.

The work covered in this handover mainly focuses on:

- authentication protection
- input validation
- input sanitisation
- rate limiting
- IP throttling
- scraper protection
- abnormal traffic monitoring
- honeypot endpoint logging
- review of penetration testing findings

## Future Work

The penetration testing report should be used as a key reference for future work. Some findings have already been partly addressed, while other items such as CORS, HTTPS, Content Security Policy headers, Swagger exposure, pagination limits, and further access control improvements should be continued in the next study period.
